🧠 SupervisorAgent Opus 4
Orchestrateur principal. Coordonne les agents, gère les dépendances, génère les rapports, soumet les décisions critiques à validation humaine.
launch_agent() get_agent_status() read_state() request_human_validation() generate_report() rollback_step()
📚 MatieresAgent Sonnet 4
Normalise les 3 766 matières uniques, résout les 557 groupes de doublons, crée le référentiel canonique.
read_class_fields() fuzzy_match() get_matiere_context() propose_merge() write_canonical()
🏛️ UEAgent Sonnet 4
Réorganise les UE, comble les 47% de champs vides, normalise les codes, gère la hiérarchie UeSup → Ue → Matiere.
read_teaching_units() parse_ue_string() infer_ue_assignment() validate_ue_hierarchy()
📅 PeriodesAgent Sonnet 4
Normalise les périodes, détecte les anomalies (>4 périodes/an), crée les entités Session + Semestre + Periode.
read_periods() detect_anomaly() check_period_usage() map_to_schoolart_period()
📝 NotesAgent Sonnet 4
Valide les 52 548 fichiers de notes, détecte les notes uniformes suspectes, mappe CC/Examen vers cc1/cc2/examen.
read_notes_file() detect_uniform_grades() validate_grade_range() resolve_student_identity()
👩‍🎓 ElevesAgent Sonnet 4
Déduplique les 55 925 étudiants, normalise les nationalités (331→52), complète les champs, gère l'héritage User → Etudiant.
read_pupils() find_duplicates() normalize_nationality() generate_login() map_to_etudiant()
🔄 TransformAgent Sonnet 4
Génère les payloads API Platform conformes (JSON-LD), gère les IRI, pilote l'injection REST vers SchoolArt.
build_api_payload() call_api() resolve_iri() batch_insert() verify_insertion()

📆 Plan de création — 5 phases

Phase 1 : Infrastructure
Semaine 1
Setup projet Node.js + Claude Agent SDK • Outils MCP partagés (file-reader, normalizer, fuzzy-match) • Client API SchoolArt (JWT, CRUD, IRI) • State-manager + checkpoints • Scripts Layer 1 (nettoyage déterministe)
Phase 2 : Agents de nettoyage
Semaines 2–3
MatieresAgent (normalisation + fusion + référentiel) • UEAgent (hiérarchie + complétion) • PeriodesAgent (normalisation + anomalies) • ElevesAgent (dédup + nationalités) • NotesAgent (validation + nettoyage) • Tests unitaires sur échantillon de 5 classes
Phase 3 : Orchestration
Semaine 4
SupervisorAgent (orchestration + dépendances) • Interface validation humaine (CLI) • Génération automatique de rapports • Intégration dashboard (onglet Migration) • Test pipeline complet sur 10 licences pilotes
Phase 4 : Injection
Semaine 5
TransformAgent (payloads API Platform + IRI) • Endpoints batch sur SchoolArt si nécessaire • Test d'injection sur staging • Validation données injectées (comptages + intégrité)
Phase 5 : Production
Semaine 6
Exécution complète 65 licences (production) • Rapport final de migration • Vérification post-migration (cohérence, intégrité référentielle) • Documentation technique et handover
